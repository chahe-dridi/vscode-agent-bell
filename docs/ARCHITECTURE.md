# Agent Bell — Architecture

## How it works

```
VS Code terminal (shell integration active)
        │
        │  onDidStartTerminalShellExecution
        ▼
  watchExecution()  ← async generator, streams chunks
        │
        │  per chunk
        ▼
  maybeTrigger()
    ├─ watching? (global toggle)
    ├─ terminalPassesNameFilter()?
    ├─ stripAnsi(chunk) → clean text
    ├─ getPatterns() → cached RegExp[]
    ├─ debounce check (lastTriggerAt map)
    └─ playSound()
          ├─ macOS   → afplay -v <vol> <file>
          ├─ Windows → powershell SoundPlayer.PlaySync()  (NOT detached)
          └─ Linux   → paplay --volume | aplay
```

## Key design decisions

**Shell integration required.** VS Code's `onDidStartTerminalShellExecution` only fires for terminals where the shell has integration active (bash, zsh, fish, PowerShell). Raw shells, WSL without integration, or full-screen TUI apps (ncurses) won't expose output. Nothing we can do here — it's a VS Code API constraint.

**Pattern cache.** Regex compilation is expensive per-chunk. Patterns are compiled once into `cachedPatterns` and invalidated only when `agentConfirmSound.patterns` changes in settings.

**No `detached: true` on Windows.** Detached processes on Windows run in a new process group and can lose access to the audio device. macOS and Linux use `detached: true` so the sound process outlives the spawn and can play to completion.

**Per-terminal debounce.** `lastTriggerAt` is a `Map<Terminal, number>`. Each terminal tracks its own last-trigger timestamp independently. The map is cleaned on `onDidCloseTerminal` to avoid memory accumulation.

**ANSI stripping.** Terminal output comes in with escape sequences (colors, cursor moves, etc). `stripAnsi` removes them before pattern matching so regexes work on plain text.

## File structure

```
agent-confirm-sound/
├── src/
│   └── extension.ts        ← all extension logic (single file, keep it that way)
├── media/
│   └── notify.wav          ← bundled default sound
├── out/                    ← compiled JS (gitignored, generated by tsc)
├── dist/                   ← packaged .vsix output (gitignored)
├── docs/
│   ├── ROADMAP.md          ← features, fixes, enhancements
│   └── ARCHITECTURE.md     ← this file
├── icon.png                ← marketplace icon (128×128)
├── package.json
├── tsconfig.json
├── .vscodeignore           ← keeps .vsix lean
└── .gitignore
```

## Settings namespace

All settings live under `agentConfirmSound.*`. The VS Code config key is camelCase, display name is "Agent Bell".
